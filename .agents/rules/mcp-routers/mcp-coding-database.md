# Workflow: MCP Coding & Database Router

Load when: The user requests operations related to source control, code analysis, databases (SQL/NoSQL), algorithmic problem solving, or OpenAPI integrations.

## Available MCPs in this category:
- `github` & `git`: Repo reading, PR creation, version control.
- `gh_grep`: Ultra-fast code grep search.
- `codegraph`: Source code structure and call graphs.
- `sqlite`: Local SQLite database queries.
- `excel-db`: SQL queries on .xlsx files.
- `leetcode`: Fetch, solve, and submit LeetCode problems.
- `stackoverflow`: Direct StackOverflow solution search.
- `sequential-thinking`: Step-by-step logic framework for complex problems.
- `mcp-jupyter`: Jupyter Notebook connection for Python/graphs.
- `hey-daddy`: Task management and code quality/syntax checker.
- `recon-wrxn`: Blast radius and call graph analysis.
- `storybook`: Read and test Storybook UI components.
- `openapi`: Convert Swagger/OpenAPI specs into interactive API tools.
- `redis`: Local Redis RAM/Cache database management.
- `verigen`: Verilog/RTL coding, code generation, and testbench creation.
- `mysql`: MySQL database management and SQL optimization.

## Usage Guidelines:
1. Determine if the task involves local codebase or external repos/databases.
2. For databases, always verify connection strings and use read-only queries first for safety.
3. For algorithmic/complex logic, invoke `sequential-thinking` before writing code.
